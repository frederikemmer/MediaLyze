# Contributing

## Development

- simple use the docker-compose to build local

## Expectations

- Keep modules focused and typed.
- Add or update tests for changed logic.
- Preserve read-only behavior for media files.
- Document changes in `/docs`.

## Pull Requests

- Keep scope focused.
- Describe user-visible behavior changes.
- Do not include secrets or machine-specific artifacts.
